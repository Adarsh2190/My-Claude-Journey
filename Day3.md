Role-Based Prompting

Question Used

**"How can I improve user engagement in my mobile app?"**

---

1. Without Assigning Any Role

Prompt

```
How can I improve user engagement in my mobile app?
```

Output

* Improve app design and user experience.
* Send notifications to users.
* Add rewards and incentives.
* Personalize content.
* Collect user feedback regularly.


2. Using Founder Persona

Prompt

```
You are a startup founder.

How can I improve user engagement in my mobile app?
```

Output

* Focus on retention metrics such as DAU/MAU.
* Identify the core value users receive.
* Build engagement loops and referral systems.
* Use customer interviews to understand drop-offs.
* Prioritize features that directly impact business growth.


3. Using Developer Persona

Prompt

```
You are a senior software developer.

How can I improve user engagement in my mobile app?
```

Output

* Optimize app performance and loading speed.
* Implement push notification services.
* Use analytics tools to track user behavior.
* Introduce gamification features.
* A/B test new features and UI changes.
* Fix bugs and reduce crashes.


Comparison of Responses

| Prompt Type       | Focus                                               |
| ----------------- | --------------------------------------------------- |
| No Role           | General suggestions                                 |
| Founder Persona   | Business growth, retention, revenue, strategy       |
| Developer Persona | Technical improvements, performance, implementation |

Key Observation

The same question produced different answers because Claude adapted its perspective based on the assigned role.


Learnings

1. Role-Based Prompting changes the AI's perspective.
2. Different roles produce different types of solutions.
3. Founder personas focus on business outcomes.
4. Developer personas focus on technical implementation.
5. Assigning a role helps generate more targeted and relevant responses.
6. Role-Based Prompting is useful for obtaining expert-like insights from different domains.
