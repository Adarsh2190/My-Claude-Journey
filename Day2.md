What I Learned from This Prompt Engineering Exercise

1. AI is only as good as the instructions I provide

A vague prompt produces vague results.

Example:

> "Write about dogs."

Output:

> Generic information about dogs.


2. Specific prompts create targeted outputs

Example:

> "Write 3 fun facts about Border Collies for a kids newsletter. Use simple words."

Output:

> Focused, audience-specific, useful content.

3. A great prompt contains four key elements

| Element | Purpose                             |
| ------- | ----------------------------------- |
| Role    | Who the AI should act as            |
| Task    | What the AI should do               |
| Context | Background information              |
| Format  | How the answer should be structured |

Example:

> Act as a senior data analyst.
> Analyze sales data.
> Compare 2024 and 2025 revenue.
> Present findings in a table.


4. Prompt Engineering is a skill, not a trick

I learned techniques such as:

* Being specific
* Defining audience
* Setting constraints
* Giving examples
* Assigning roles
* Refining prompts iteratively

5. Better prompts save time

Instead of:

> "Explain Power BI"

Use:

> "Explain Power BI relationships for a beginner data analyst with examples and interview questions."

This reduces follow-up questions and improves output quality.
